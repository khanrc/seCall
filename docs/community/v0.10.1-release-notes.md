## v0.10.1 (P44)

- `secall wiki update` now attempts an automatic vault `git pull --rebase` at startup to support multi-machine wiki sync.
- When the same wiki page is updated on different hosts and a git conflict occurs under `wiki/*.md`, seCall regenerates the page from the union of both sides' `sources`.
- Added `--no-pull` for offline or manually managed sync flows.
- `merge_with_existing()` no longer concatenates old and new bodies; the latest generated body replaces the old one while keeping the `sources` union.
- Scope is limited to wiki pages. Daily logs, raw session files, and graph outputs still require manual conflict handling if they collide.
