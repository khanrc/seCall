# P48 — P46/P47 신규 경로 회귀 테스트 보강

## Description

P46(Ollama Cloud graph/log 전환)과 P47(CoreML EP + embedding cloud)에서 다수의 신규 dispatch arm, env propagation, REST 마스킹 경로가 추가되었지만, code-review-graph 보고에서 다음 함수들이 untested:

- `extract_with_ollama_cloud`, `extract_with_openai_compat`, `make_fm`, `make_fm_with_summary` — graph semantic 신규 경로 (P46)
- `wiki::ollama::OllamaBackend::generate` 의 `bearer_auth` 분기 — P46
- `create_vector_indexer` 의 `"ollama_cloud"` arm — P47
- `generate_log_body` 의 `"ollama_cloud"` arm HTTP 호출 — P46

P48 은 이 신규 경로에 회귀 보호망을 부착해, 다음 plan(추가 백엔드 / dispatch 손댈 가능성 높음)이 회귀를 일으키지 않도록 합니다.

## Expected Outcome

- P46/P47 신규 dispatch / env / REST 경로에 단위/통합 테스트 부착
- code-review-graph 의 P46/P47 untested 핵심 함수 → tested
- mockito 기반 HTTP 요청 헤더/엔드포인트 회귀 검증 강화

## Subtasks

1. **graph::semantic ollama_cloud + openai_compat 분기 단위 테스트** (`task-01`, parallel_group=1)
2. **log backend resolve ollama_cloud 분기 보강 + generate_log_body 회귀** (`task-02`, parallel_group=1)
3. **wiki::ollama api_key bearer_auth 분기 mockito 테스트** (`task-03`, parallel_group=1)
4. **REST API embedding/sessions 라우터 회귀 보강** (`task-04`, parallel_group=2)
5. **vector dispatch ollama_cloud arm fallback 테스트** (`task-05`, parallel_group=2)

## Constraints

- **기존 통과 테스트 수정 금지** — 새 테스트만 추가
- **단위 테스트 우선** — 통합 테스트는 라우터 수준에서만
- **mockito 사용 일관성** — 기존 `wiki/ollama.rs` 패턴 그대로
- **ENV_MUTEX / tokio sync** — env 사용 테스트는 기존 mutex 패턴 유지

## Non-goals

- code-review-graph 의 untested 249건 전부 — P46/P47 신규 경로 한정
- 새 기능 추가 (백엔드 추가, dispatch 분기 추가 등)
- UI 테스트 (P47 에서 SettingsRoute.test.tsx 9건 추가로 충분)
- 결과 보고서 `docs/plans/p48-p46-p47-result.md` 작성 — tunaFlow 자동 생성
