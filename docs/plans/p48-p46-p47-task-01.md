## Change description

### 단계별 접근

1. **mockito server 패턴 재사용** — 기존 `test_extract_with_llm_ollama_uses_config_model` 의 setup 코드를 참조해 동일 구조로 작성.

2. **함수 단위 호출** — `extract_with_llm` 를 거치지 않고 직접 `extract_with_ollama_cloud` / `extract_with_openai_compat` 를 호출 (둘 다 `pub(crate)` 가시성 — 같은 모듈 내 테스트에서 호출 가능).

3. **헤더 검증** — mockito 의 `mock.with_request_header_check(...)` 또는 `Mock::matched_n_times` 패턴으로 `Authorization` 헤더 검증.

4. **에러 경로** — 401 / invalid JSON 반환 시 `Err` 반환 확인.

### 구현 제약

- **테스트는 모듈 내부 `mod tests`** — extern test crate (`tests/` 디렉토리) 가 아님. 함수 가시성 문제 회피.
- **mockito v1 사용** — 이미 dev-dependency 에 있음 (`crates/secall-core/Cargo.toml:59`).
- **각 테스트 독립** — env 의존 없음, ENV_MUTEX 불필요.
