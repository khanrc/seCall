## Change description

### 단계별 접근

1. **embedding PATCH 정상 경로** — non-secret 필드 갱신이 sanitize 로직에 막히지 않는지 확인. P47 의 sanitize 가 `cloud_api_key` key 만 차단하는데, 실수로 `backend` 등을 차단할 가능성 회귀 보호.

2. **GET 응답 schema 검증** — pool_size, cloud_host 가 serde 직렬화 시 정확한 타입(number, string)으로 나오는지.

3. **env_indicators 확장** — ANTHROPIC 만 검증되어 있어 OLLAMA_CLOUD_API_KEY 가 빠지면 회귀.

4. **sessions 라우터 (optional)** — 시간 부족 시 1번만.

### 구현 제약

- **ENV_MUTEX (TokioMutex) 사용** — 기존 `rest_config.rs:12` 패턴 그대로.
- **tempdir + SECALL_CONFIG_PATH override** — 기존 setup 헬퍼 그대로 재사용.
- **단순 status + json body assert** — 라우터 회귀이므로 복잡한 setup 회피.
