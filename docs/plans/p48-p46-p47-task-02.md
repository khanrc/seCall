## Changed files

### 기존 테스트 보강

- `crates/secall/src/commands/log.rs` 의 `#[cfg(test)] mod tests` (line 440+) 에 다음 테스트 추가:

  1. **`test_resolve_log_api_url_ollama_cloud_uses_log_cloud_host_first`**
     - `config.log.cloud_host = Some("https://custom.ollama.example")` 설정
     - `resolve_log_api_url(&config, "ollama_cloud")` 가 정확히 `"https://custom.ollama.example"` 반환
     - cloud_host 미설정 시 `graph.cloud_host` 로 폴백, 그것도 없으면 `https://ollama.com` default 폴백 — 3가지 케이스 각각 케이스

  2. **`test_resolve_log_api_url_ollama_cloud_falls_back_to_graph_cloud_host`**
     - `config.log.cloud_host = None`, `config.graph.cloud_host = Some("https://graph.ollama.example")`

### 신규 통합 테스트

- `crates/secall/tests/log_backend_resolve.rs` 에 `generate_log_body` 의 ollama_cloud arm 통합 회귀 추가:

  3. **`generate_log_body_ollama_cloud_includes_bearer_auth`** (async, tokio)
     - mockito server `/api/chat` POST 200 응답 모킹
     - `config.log.backend = "ollama_cloud"`, `config.log.cloud_api_key = Some("env-key")`, `config.log.cloud_host = Some(&server.url())`
     - `generate_log_body(&config, &context, None /* cli_model */).await` 호출
     - mockito 가 받은 요청 헤더에 `Authorization: Bearer env-key` 포함 확인

  4. **`generate_log_body_ollama_cloud_uses_resolve_log_model_chain`**
     - log.cloud_model 없이 graph.cloud_model 만 설정 → 요청 body 에 graph.cloud_model 사용 확인
     - 둘 다 없으면 `LOG_OLLAMA_CLOUD_DEFAULT` (`kimi-k2.6:cloud`) 사용

  5. **`generate_log_body_ollama_cloud_missing_api_key_returns_err`**
     - `cloud_api_key = None` 상태에서 호출 → `Err` 반환, 메시지에 `OLLAMA_CLOUD_API_KEY` 또는 유사 단어 포함

## Change description

### 단계별 접근

1. **log.rs 내부 테스트 — resolve_log_api_url 빈틈** — 현재 ollama_cloud arm 의 cloud_host fallback 체인(log.cloud_host → graph.cloud_host → default) 이 명시 테스트 안 됨. 3 케이스 추가.

2. **tests/log_backend_resolve.rs — generate_log_body 통합** — mockito + tokio runtime. `generate_log_body` 가 async 라 `#[tokio::test]` 필수. 기존 `log_backend_resolve.rs` 가 `mockito` 의존 안 했으면 dev-dependency 확인 — `crates/secall/Cargo.toml` 의 dev-dependencies 에 mockito 추가 (만약 없으면).

3. **api_key 없음 → Err 경로** — secret 보호 측면에서 중요. P46 Rework 1 에서 이미 sanitize 했지만 실제 사용 시 호출 거부도 검증.

### 구현 제약

- **`generate_log_body` 가 `pub` 인지 확인** — `pub fn` 이 아니면 `pub(crate)` 로 변경하거나 wrapper 통해 호출. 기존 `pub fn resolve_log_model` 처럼 publication 통일.
- **mockito server URL 으로 cloud_host override** — 실제 ollama.com 호출 안 함.
- **ENV_MUTEX 불필요** — env 미사용 (config 객체 직접 조작).
