# P48 Task 03 — wiki::ollama api_key bearer_auth 분기 mockito 테스트

## Changed files

### 단위 테스트 추가

- `crates/secall-core/src/wiki/ollama.rs` (현재 `pub api_key: Option<String>` 분기는 `generate()` 함수 내부 `if let Some(key) = &self.api_key { req = req.bearer_auth(key); }` 한 줄 — line 28-29) 에 `#[cfg(test)] mod tests` 신규 또는 기존 모드에 다음 추가:

  1. **`test_ollama_backend_generate_includes_bearer_auth_when_api_key_set`** (async)
     - mockito server `/api/generate` 또는 `/api/chat` (실제 endpoint 확인 — wiki 생성 시 `generate` 함수가 사용하는 path) 200 응답 모킹
     - `OllamaBackend { api_url: server.url(), model: "gemma4:31b-cloud", max_tokens: 4096, api_key: Some("cloud-key") }` 인스턴스로 `generate("prompt").await` 호출
     - mockito 가 받은 요청 헤더에 `Authorization: Bearer cloud-key` 포함 검증

  2. **`test_ollama_backend_generate_omits_auth_header_when_api_key_none`** (async)
     - 같은 setup, `api_key: None`
     - mockito 가 받은 요청 헤더에 `Authorization` 헤더 **없음** 검증 (local Ollama 호출 시 인증 안 보냄)

  3. **`test_ollama_backend_generate_propagates_4xx_error`** (async, optional)
     - mockito 가 401 반환 → `generate` 가 `Err` 반환

### 기존 호출자 호환성

- `crates/secall/src/commands/wiki.rs` 의 `OllamaBackend { ..., api_key: None }` 사용 (P46 에서 추가) 는 그대로 둠. 본 task 는 테스트만 추가.

## Change description

### 단계별 접근

1. **wiki/ollama.rs 의 generate 함수 endpoint 확인** — `/api/chat` 인지 `/api/generate` 인지 코드 확인 후 mockito mock path 일치시킴. (P46 작업으로 이미 알고 있지만 작업 시 정확히 확인)

2. **mockito server 띄워 두 분기 검증** — api_key Some / None 각각.

3. **에러 경로 (optional)** — 4xx 응답 시 명확한 Err 메시지.

### 구현 제약

- **wiki/ollama.rs 에 `mod tests` 없을 수 있음** — 신규 생성. `#[cfg(test)]` 가드 필수.
- **async 테스트** — `#[tokio::test(flavor = "multi_thread", worker_threads = 2)]` 사용. mockito 가 sync 가 아닌 비동기 호출 받음.
- **mockito v1** — 이미 `secall-core/Cargo.toml:59` 에 dev-deps 로 존재.
