### 5.1 모듈 역할/경계 매트릭스

### 5.2 위험도별 발견

**HIGH**
- `crates/secall/src/commands/graph.rs:364, 396` — `Database::open().unwrap()` 런타임 패닉 가능. CLI 사용자가 보는 첫 에러.
- `graph/semantic.rs:115–289` — Anthropic / Ollama / Ollama-Cloud / OpenAI-compat **4개 백엔드를 각각 HTTP 호출로 직접 구현** → 요청/응답/타임아웃/에러 패턴이 거의 동일하게 중복. `WikiBackend` 처럼 trait 추출 필요.
- `mcp/server.rs:465–510` (config PATCH) — `cloud_api_key` 필터가 graph/log/embedding 섹션에만 적용. **wiki 섹션은 현재 필드가 없어도 선제 sanitize 추가 권장** (메모리 `feedback_rest_api_security` 정합).

**MED**
- `crates/secall/src/commands/wiki.rs:167` `run_update_with_sink` 404 lines — 파싱/임베딩/저장 단일 함수. 분해 권고.
- `crates/secall/src/commands/ingest.rs:2627` `ingest_sessions` 368 lines — 경로 수집/분류/임베딩/병렬화 혼재.
- `store/db.rs:177` `Database::conn() -> &Connection` public — 현재 사용처는 모두 store 내부지만 외부 raw SQL 우회 가능. `pub(crate)` 강등 검토.
- `ingest.rs pattern_rules()` 내 `Regex::new(p).unwrap()` — 하드코딩 테스트 패턴은 정당. user config 경로는 `classify.rs` 처럼 `.map_err()` 처리 일관화 권고.
- REST API 테스트 갭 — `PATCH /api/config/{section}`, `POST /api/sessions/{id}/{tags,favorite,notes}` integration 테스트 부재. (P48 에서 일부 보강됨 — 잔여 갭 확인 필요)

**LOW**
- Regex pre-compile 정책 — core(`graph/extract.rs`)는 `OnceLock + get_or_init` 적용 (P18 Rev.2 정합).
- CLI dispatch — clap subcommand + 균일한 `pub async fn run()`. `NoopSink` 로 CLI/REST progress sink 분리 우수 (P36).
- Error 타입 — `secall-core::Result<T>` (SecallError) ↔ `anyhow::Result<T>` (CLI) 2-층 분리, `?` 자동 변환.
- DTO 누수 — REST/MCP 모두 store 모델 미노출.
- CORS / vault traversal — 로컬 전용 + relative path join, 수용 가능.
- Jobs adapters — `Box<dyn Fn ... + Send + Sync>` 로 secall→core 역의존 회피.
